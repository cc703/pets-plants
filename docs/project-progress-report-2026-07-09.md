## 6. 当前仍需注意的问题

### 6.1 项目源码 Git 状态

当前 `app/`、`server/`、`src/`、`scripts/`、`docs/` 等主体目录在 Git 状态中显示为未跟踪文件。这说明当前项目可能还处在本地搭建或迁移阶段，尚未完成一次规范提交。

建议下一步在确认代码可运行后，整理一次提交：

- 先确认不需要提交的本地文件，例如 `.env`、缓存、构建产物。
- 确认 `.gitignore`。
- 将当前可运行版本作为一个基线提交。

### 6.2 邮箱找回密码尚未完整实现

目前为了安全，邮箱重置密码路径已临时关闭。后续如要开放，需要补齐：

- 邮件验证码或邮件链接 Token。
- Token 过期时间。
- Token 一次性使用。
- 发送频率限制。
- 前端邮箱找回流程提示。

### 6.3 限流为内存版本

当前限流中间件适合本地开发和单进程演示。若进入生产环境，需要迁移到 Redis 或网关层限流，否则多进程或重启后限流状态会丢失。

### 6.4 AI 建议仍需内容安全策略

当前已经增加“不能替代兽医诊疗”的提示，但如果后续面向真实用户，需要继续增加：

- 危急症状识别。
- 敏感用药建议拦截。
- 明确就医建议。
- 回复内容审查。
- AI 调用失败、超时、限流的前端提示。

### 6.5 商城仍是虚拟兑换模型

当前积分商城是虚拟道具、主题和展示权益兑换，不是真实商品电商。若后续要做真实商城，需要新增：

- 商品库存。
- 订单。
- 地址。
- 支付。
- 售后。
- 物流。
- 风控。

### 9.2 邮箱找回密码

已实现邮箱一次性 Token 重置密码链路：

- 新增 `email_reset_tokens` 表。
- 注册时支持保存邮箱。
- 新增 `POST /api/auth/email/reset/send`。
- 邮箱重置 Token 使用 bcrypt hash 存储。
- Token 有效期 15 分钟。
- 生成新 Token 时自动作废旧 Token。
- Token 使用后标记为已使用。
- 重置成功后删除用户所有 refresh token。
- 非生产环境返回 `debugToken`，方便本地 smoke 测试。
- 生产环境不返回明文 Token，后续可接真实邮件服务。

### 9.3 生产级安全加固起步

- 数据库不可用时自动退回内存限流。
